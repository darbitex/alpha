Darbitex alpha is retired.

Shut down 2026-04-10. All liquidity withdrawn.

The alpha deployment held its package upgrade authority in a single wallet. On Aptos, a compatible upgrade can rewrite function bodies — which means one key could override the admin multisig, drain reserves, or add a rug entry. Multisig admin without multisig publisher is security theater.

There is no in-place fix. Upgrade authority is bound to the publisher account at publish time, and cannot be transferred. The fix is a clean redeploy from a multisig publisher account — so from day one, upgrading the package requires the same threshold as changing the admin.

A successor is being built. The contracts are essentially the same. What changes is who holds the key.
